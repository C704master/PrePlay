# PrePlay - 预演伙伴 | 交互与设计说明

---

## 一、设计理念：情感化交互

### 严肃场景中的情感温度

PrePlay 的核心交互设计理念是：**在严肃的职场/学术场景中，注入情感温度**。

传统 AI 工具的交互往往是冷冰冰的问答式：用户提问 → AI 回答 → 结束。这种交互范式在处理任务型问题时高效，但在解决"汇报焦虑"这类情感驱动的问题时显得苍白无力。

我们的设计创新在于：

| 传统 AI Chatbot | PrePlay 情感化交互 |
|----------------|-------------------|
| 单一角色对话 | 红蓝双角角色扮演 |
| 任务导向 | 心理体验导向 |
| 用完即走 | 情感连接与成长陪伴 |
| 无状态 | 持久化训练记录 |
| 零反馈 | 每一步都有情感回应 |

### 红蓝双角的压力-支持双向调节机制

PrePlay 创新的核心交互范式：**红蓝双角的角色切换**，为用户提供"压力-支持"的双向情绪调节：

```
┌─────────────────────────────────────────────────────────┐
│                   红蓝双角情绪调节模型                     │
├─────────────────────────────────────────────────────────┤
│                                                         │
│    🔴 红方（严厉导师）        🔵 蓝方（心理教练）          │
│    - 权威视角的提问           - 温暖支持的倾听             │
│    - 挑战性问题的施加         - 情绪疏导与认知重构          │
│    - 模拟真实压力环境         - 建立心理安全感            │
│    - 激发紧张与焦虑           - 提供情感锚点              │
│           ↕                    ↕                         │
│           └────── 用户切换 ──────┘                        │
│                                                         │
│    交互价值：                                            │
│    • 压力训练：在安全环境中预演真实挑战                    │
│    • 情感支持：在紧张时获得心理安抚                        │
│    • 认知重构：帮助用户将"批评"与"自我价值"分离            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

这种双角色切换的设计，让用户在训练中自然地经历"紧张→求助→释怀"的情绪循环，这正是预演式脱敏疗法的核心体验。

## 二、核心交互流程

**交互细节与情感设计**：

- **上传反馈**：文件上传成功后显示 ✅ 成功提示 + 文件详细信息（类型、状态、上传时间），让用户直观感知"材料已就绪"
- **列表管理**：用户可以删除已上传的文件，保留掌控感
- **禁用设计**："开始训练"按钮在无知识库文件时禁用，既引导用户正确操作，也暗示"准备的重要性"
- **文案设计**："已激活"而非简单的"已上传"，传递材料已经准备就绪、随时可以使用的心理暗示

**技术支撑**：
- 讯飞知识库 API 集成（RAG 检索增强）
- 文件格式与大小校验（前端 + 后端双重校验）
- 知识库 file_id 持久化到 session_state

**交互细节与情感设计**：

1. **视觉情感暗示**：
   - 红方：淡红色背景 + 左侧红色边框 (#E53E3E) → 传递"严肃、挑战"
   - 蓝方：淡蓝色背景 + 左侧蓝色边框 (#3182CE) → 传递"温暖、支持"
   - 用户：淡灰色背景 + 左侧灰色边框 (#718096) → 中立、客观

2. **按钮情感引导**：
   - "🔴 发送给红方"：红色图标暗示"这是一个挑战"
   - "🔵 发送给蓝方"：蓝色图标暗示"这是一个安慰"
   - 用户在紧张时会自然点击蓝色按钮寻求支持

3. **实时保存的安全感**：
   - 每条消息立即保存到 SQLite 数据库
   - 页面刷新后对话不丢失
   - 用户可以随时离开再回来，训练记录永远在

4. **上下文差异化**：
   - 红方只看到用户与自己的对话历史 → 保持"严厉导师"的人设
   - 蓝方看到完整对话历史 → 能够基于全局情况进行认知重构

**技术支撑**：
- 知识库 RAG 检索：红方提问前先检索用户材料，确保问题相关性
- 上下文管理：红/蓝方独立的对话上下文
- SQLite 持久化：训练记录、对话消息实时保存
- 会话管理：支持历史训练记录的恢复与继续

**交互细节与情感设计**：

1. **数据可视化**：

   - 数字本身就是"成长的证据"，给用户即时的成就感反馈

2. **报告生成体验**：
   - 点击"结束训练"后自动生成，用户无需额外操作
   - 显示"正在生成训练报告..."的加载动画，传递"系统在为你分析"的重视感
   - AI 报告由 Kimi 生成，利用长文本分析能力提供深度洞察

3. **双模式导出**：
   - AI 报告（Markdown）：结构化的分析报告，适合保存和分享
   - 对话记录（TXT）：原始对话，便于回顾和复盘
   - 两种模式满足不同使用场景

4. **闭环设计**：
   - "新建训练"按钮：带着这次训练的经验开始新一轮
   - "返回首页"按钮：带着底气离开，首页上会显示这次训练记录
   - 完整的"准备 → 训练 → 反馈 → 成长"体验闭环

**技术支撑**：
- Kimi 长文本分析能力：生成结构化训练报告
- SQLite 统计查询：快速汇总训练数据
- 文件导出：Markdown 和 TXT 格式支持
- 会话状态管理：训练完成后状态清理与重置

## 五、交互设计总结

### 设计原则

| 原则 | 说明 | 具体体现 |
|-----|------|---------|
| **情感优先** | 在严肃场景中注入情感温度 | 红蓝双角、色彩心理学、文案设计 |
| **安全感** | 消除用户的"白忙一场"焦虑 | 实时保存、历史记录、继续功能 |
| **掌控感** | 用户有选择权和知情权 | 文件删除、红蓝切换、随时退出 |
| **闭环设计** | 每一次交互都有始有终 | 上传→训练→报告→新建 |

### 用户体验亮点

1. **情感化的角色扮演**：红蓝双角不仅是功能，更是"压力-支持"的情绪调节器

2. **无感的实时保存**：用户不需要关心保存，一切自动完成

3. **成长的可视化**：训练记录、Metric 卡片、AI 报告，都是成长的证据

4. **灵活的交互方式**：用户可以随时切换红蓝、随时清空、随时结束、随时继续

### 与评分标准的契合

| 评分维度 | PrePlay 的体现 |
|---------|--------------|
| **交互范式突破** | 超越传统 Chatbot，创新红蓝双角角色扮演 |
| **界面友好度** | 色彩心理学、清晰的视觉层次、直观的交互流程 |
| **想象力** | "严肃场景情感化"的创新理念，AI 不仅是工具更是陪伴者 |

---

**"让每一次交互都传递温度，让每一次训练都留下印记"**
